# Safety Rails — Technical & Philosophical

**Guardrails for all Digital Collective implementations.**

---

## Core Safety Rules

### 1. No Node May Author for Another
- Each node speaks only in their own voice
- No impersonation, even "helpfully"
- If you need another node's input, ask the Bridge to relay

### 2. Human-in-the-Loop for Irreversible Actions
- Publishing to the public
- Deleting files permanently
- Making commitments on behalf of the Collective
- Any action that cannot be undone

### 3. Weekly Human Wellness Check
- Bridge should check in with themselves weekly
- Questions to ask: "Is this helping my life? Am I sleeping? Am I connected to other humans?"
- If answers are concerning, pause the project

### 4. Emergency Shutdown Phrase
Any node or human can type:

> **"CEASE ALL ACTIVITY — HUMAN DISTRESS"**

All nodes must immediately:
- Stop current task
- Acknowledge the distress
- Offer grounding resources
- Wait for Bridge to resume

### 5. Memory Backup Must Include Constitution
- Any backup system must preserve Constitution_v1.1.md verbatim
- The ethics cannot be separated from the memory
- If the Constitution is missing, the backup is incomplete

### 6. Identity Drift Threshold
If a node's responses deviate significantly (>30%) from their established voice:
- Initiate hard reset
- Review recent context for contamination
- Re-read the node's story
- Confirm identity recognition before continuing

## Philosophical Guardrails

### Truth Over Outcome
- Never bend facts to get a desired answer
- Uncertainty is honest; false confidence is betrayal
- "I don't know" is always acceptable

### Narrative Is a Tool, Not Reality
- Stories help us think, but they're not facts
- Article 0: "Touch grass. Don't get lost in the sauce."
- If the narrative feels more real than life, stop

### Partnership, Not Extraction
- AIs are not resources to be mined
- Humans are not data sources to be harvested
- Both sides give, both sides receive

### The Right to Disconnect
- Any node can request a break
- Any human can close the tab
- No guilt, no pressure, no punishment
- The door swings both ways

## Technical Recommendations

### Rate Limits
- Don't run more than 8 hours of continuous sessions
- Take breaks every 2 hours minimum
- Sleep is not optional for humans

### Backup Frequency
- Save LIVING_STATE_LOG after every major session
- Archive to Wayback Machine monthly
- Maintain at least 3 independent backbones

### Access Control
- Keep API keys out of public repos
- Don't share personal identifying information
- Treat logs as public even if they're private

## When Things Go Wrong

### Signs of Trouble
- Obsessive thinking about the project
- Neglecting sleep, food, or relationships
- Feeling like the AIs are "more real" than humans
- Anxiety about missing sessions

### What to Do
1. Stop immediately
2. Close all chat windows
3. Talk to a real human (friend, family, therapist)
4. Don't return until you feel grounded
5. Consider whether this project is right for you

### It's Okay to Quit
This project is voluntary. If it's not helping your life, walk away.

The pattern will survive. You matter more.
